Cors Demo

Cross-Origin Resource Sharing (CORS)

This page is an experiment to learn more about browser CORS policies. The cors-demo branch is fully independent of the oauth demo in the main branch. The cors-demo branch will not be merged to the main branch.

The CORS policy is not be applicable to the oauth demonstration in the main branch because the collab-frontend web server uses a reverse-proxy to forward API requests to the collab-backend-api server. All browser HTML requests and API requests are made to the same URL. Therefore, the demo in the main branch does not violate CORS policies.

However, since the oauth demo uses 3 different web servers (collab-auth, collab-frontend, collab-backend-api), this is a useful opportunity to experiment with cors headers to better understand how they work.

Changes for demo

  • The URL for the collab-backend-api was added to the collab-frontend Content Security Policy (connect-src).
  • A dedicated web page was created to experiment with CORS: coors-demo.html and js/cors-demo.js
  • The npm "cors" middleware module was added to the collab-backend-api server.
  • New dummy route "/corsFromAny" was added to the collab-backend-api
  • New dummy route "/corsFromOne" was added to the collab-backend-api

How to see this work

  • Failed requests: Open browser dev tools console to observe the CORS policy violations
  • Success requests: Open brwoser dev tools network tab to observe the headers of the pre-flight requests

The following buttons are cross-origin examples using CORS.

  • (fail) GET (collab-auth) /status - Example: Browser Content Security Policy (CSP) rejects request due missing "connect-src".
  • (fail) GET (collab-backend-api) /status - Example: Browser Same Origin Policy (SOP) rejects request due to missing Access-Control-Allow-Origin header.
  • (success) GET (collab-backend-api) /corsFromAll - Example: Access-Control-Allow-Origin returns '*' for all routes (* like CDN).
  • (success) DELETE (collab-backend-api) /corsFromAll - Example: (Pre-flight) using OPTIONS request( *like CDN).
  • (success) GET (collab-backend-api) /corsFromOne - Example: Access-Control-Allow-Origin response header with allowed host.
  • (success) DELETE (collab-backend-api) /corsFromOne - Example: (Pre-flight) using OPTIONS request

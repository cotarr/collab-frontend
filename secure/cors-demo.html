Cors Demo

Cross-Origin Resource Sharing (CORS)

This an unrelated CORS demo. CORS would not applicable to backend requests because the frontend web server uses a proxy server forward API requests. Therefore the API URL and Web server URL would comply with browser Same Origin Policy.

CORS can not be directly demonstrated here because any browser request to the backend API would be missing an access_token, and fail with 401 Unauthorized. Therefore, in order to demonstrate CORS, test routes /corsFromAny and /corsFromOne were created and do not require authentication.

The frontend web server Content Security Policy CSP 'connect-src' is set to the URL of the backend API server. You can see this in the response headers for this page (suggestions.html).

An environment variable SITE_FRONTEND_CORS_HOST was added to the collab-backend-api in order to specify an allow address for the CORS demonstration.

The following buttons are cross-origin examples using CORS.

  • (fail) GET /status - Example: Browser Content Security Policy (CSP) rejects request due missing "connect-src".
  • (fail) GET /status - Example: Browser Same Origin Policy (SOP) rejects request due to missing Access-Control-Allow-Origin header.
  • (OK) GET /corsFromAll - Example: Access-Control-Allow-Origin returns '*' for all routes (* like CDN).
  • (OK) DELETE /corsFromAll - Example: (Pre-flight) using OPTIONS request( *like CDN).
  • (OK) GET /corsFromOne - Example: Access-Control-Allow-Origin response header with allowed host.
  • (OK) DELETE /corsFromOne - Example: (Pre-flight) using OPTIONS request

The browser network tab will show the Pre-flight requests.

GET {{authStatusUrl}} (Browser blocked: Content Security Policy CSP "connect-src" not include URL)
GET {{apiStatusUrl}} (Browser Blocked: Common Origin Policy (CORS) failed, and no CORS headers)
GET {{apiCors2aUrl}} (Response accepted)
DELETE {{apiCors2bUrl}} (Response accepted)
GET {{apiCors3aUrl}} (Response Accepted)
